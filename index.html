Organisasi Komputer

Mempelajari cara kerja komputer dari sisi unit operasional, hubungan antar komponen hardware, hierarki memori, antarmuka, hingga transmisi sinyal kontrol sistem.

Pengertian Sistem Komputer

Sistem komputer adalah suatu kesatuan yang terdiri atas perangkat keras (hardware), perangkat lunak (software), dan pengguna (brainware) yang bekerja sama untuk mengolah data menjadi informasi. Dalam organisasi komputer, fokus utama adalah bagaimana komponen perangkat keras saling berinteraksi untuk menjalankan proses komputasi secara efisien.

Soal & Jawaban Pembahasan

01

Uraikan komponen utama sistem komputer dan buatlah gambar blok diagram beserta fungsinya.

Sistem komputer terdiri dari tiga komponen utama: Hardware (fisik), Software (program/instruksi), dan Brainware (manusia/pengguna).

Blok Diagram Sistem Komputer (Model Von Neumann)
+------------------+ Sinyal Kendali +--------------------+ | UNIT MASUKAN | ---------------------> | CENTRAL | | (Keyboard/Mouse) | <--------------------- | PROCESSING UNIT | +------------------+ | (CPU) | | +--------------------+ | Transfer Data | v | Transfer Hasil +------------------+ v | MEMORI UTAMA | +--------------------+ | (RAM) | | UNIT KELUARAN | +------------------+ | (Monitor/Printer) | +--------------------+
  • Unit Masukan: Menerima data mentah dari luar dan mengonversinya ke sinyal digital.
  • CPU: Otak pemroses utama yang mengeksekusi perintah dan mengendalikan semua unit.
  • Memori Utama: Penyimpanan volatile untuk menyimpan program dan data aktif sementara.
  • Unit Keluaran: Menyajikan informasi hasil pemrosesan ke format yang dimengerti pengguna.
02

Jelaskan cara kerja CPU dalam memproses instruksi.

CPU bekerja menggunakan siklus berulang yang dinamakan Siklus Instruksi (Instruction Cycle):

  • 1. Fetch (Mengambil): Control Unit mengambil kode instruksi biner dari RAM berdasarkan alamat memori yang dipegang oleh Program Counter (PC).
  • 2. Decode (Menerjemahkan): Control Unit memecah dan menerjemahkan instruksi tersebut untuk mengetahui operasi apa yang harus dijalankan.
  • 3. Execute (Mengeksekusi): Komponen internal CPU (seperti ALU) menjalankan operasi hitungan atau logika sesuai perintah.
  • 4. Store (Menyimpan): Hasil akhir ditulis kembali ke dalam register atau memori utama, lalu PC diperbarui untuk instruksi berikutnya.
03

Gambarkan dan jelaskan struktur umum CPU (ALU, Control Unit, Register).

Secara internal, arsitektur dasar inti CPU terbagi menjadi tiga blok utama:

Struktur Internal CPU
+-------------------------------------------------------+ | CORE CPU | | +--------------------+ +--------------------+ | | | CONTROL UNIT | <===> | ARITHMETIC LOGIC | | | | (Pusat Kendali/CU) | | UNIT (ALU) | | | +--------------------+ +--------------------+ | | ^ ^ | | | Jalur Data | | | v v | | +-------------------------------------------------+ | | | REGISTERS | | | | (PC, IR, Accumulator, MAR, MBR) | | | +-------------------------------------------------+ | +-------------------------------------------------------+
  • Control Unit (CU): Mengatur sinkronisasi, membaca kode instruksi, dan memicu jalur kendali sirkuit.
  • Arithmetic Logic Unit (ALU): Sirkuit digital khusus yang mengeksekusi perhitungan matematika dasar (+, -, *, /) dan logika boolean (AND, OR, NOT).
  • Register: Memori internal ultra-cepat berkapasitas sangat kecil untuk menampung operand, alamat memori, dan status sistem secara instan.
04

Jelaskan hierarki memori pada komputer dan buatlah diagram piramidanya.

Hierarki memori adalah penyusunan tingkatan memori komputer berdasarkan relasi terbalik antara kecepatan akses dengan kapasitas penyimpanan.

Piramida Hierarki Memori
/\ --> Tingkat 1: Register (Dalam CPU, Tercepat, Termahal) / \ /----\ --> Tingkat 2: Cache Memory (L1, L2, L3 SRAM) / \ /--------\ --> Tingkat 3: Memori Utama (RAM / DRAM, Volatile) / \ /------------\ --> Tingkat 4: Secondary Storage (SSD, HDD, Non-Volatile) /--------------\

Semakin ke atas, memori memiliki waktu respons (latensi) paling minim namun kapasitasnya paling terbatas dan biaya produksinya tinggi.

06

Jelaskan perbedaan arsitektur RISC dan CISC.

  • RISC (Reduced Instruction Set Computer): Menekankan penggunaan set instruksi sederhana berukuran tetap yang dieksekusi dalam **satu siklus clock**. Kompleksitas dipindahkan ke perangkat lunak (compiler). Contoh: Arsitektur ARM (smartphone) dan Apple M-series.
  • CISC (Complex Instruction Set Computer): Menyediakan instruksi kompleks berskala besar yang mampu melakukan beberapa operasi memori sekaligus dalam **beberapa siklus clock**. Fokus pada kesederhanaan pembuatan program tingkat rendah. Contoh: Intel x86 dan AMD.
07

Uraikan fungsi bus data, bus alamat, dan bus kontrol beserta diagram jalurnya.

Ketiga bus ini merupakan interkoneksi utama (System Bus) penyalur sinyal listrik digital di motherboard:

  • Bus Data (Data Bus): Jalur dua arah untuk mendistribusikan data aktual antar modul.
  • Bus Alamat (Address Bus): Jalur satu arah dari CPU untuk menentukan lokasi fisik memori/port yang dituju.
  • Bus Kontrol (Control Bus): Jalur penyalur perintah operasi (Read/Write) dan sinyal waktu (Clock).
[ CPU ] <===========> BUS KONTROL (Sinyal Perintah) <===========> [ Memori / ] [ ] ------------> BUS ALAMAT (Tujuan Lokasi) ------------> [ Perangkat] [ ] <===========> BUS DATA (Isi Nilai/Bit) <===========> [ I/O ]

09

Jelaskan konsep pipelining dalam prosesor.

Pipelining adalah teknik arsitektur perangkat keras di mana prosesor memecah pengerjaan instruksi menjadi beberapa tahapan sub-proses independen, sehingga beberapa instruksi dapat diproses **secara paralel/bersamaan** pada fase siklus yang berbeda.

Sebagai analogi, ini seperti lini perakitan pabrik mobil; selagi mobil pertama memasuki tahap pengecatan, mobil kedua sudah mulai dirakit di tahap pengelasan rangka tanpa harus menunggu mobil pertama selesai total. Hal ini melipatgandakan produktivitas kinerja prosesor.

10

Jelaskan perbedaan sistem bilangan biner, desimal, oktal, dan heksadesimal.

Perbedaan mendasar terletak pada basis radiks (jumlah simbol numerik unik yang digunakan):

  • Biner (Basis 2): Menggunakan 2 simbol (0, 1). Merupakan representasi sinyal saklar digital komputer.
  • Oktal (Basis 8): Menggunakan 8 simbol (0 sampai 7). Digunakan untuk mengelompokkan kode biner 3-bit.
  • Desimal (Basis 10): Menggunakan 10 simbol (0 sampai 9). Standar hitungan peradaban manusia.
  • Heksadesimal (Basis 16): Menggunakan 16 simbol (0-9 ditambahkan A=10 s.d F=15). Efektif untuk meringkas pembacaan 1 byte data (8-bit biner menjadi 2 karakter heksa saja, misal alamat memori).
